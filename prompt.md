Task: Security + hardening pass on openclaw-root
Context
ClawMode is a multi-tenant SaaS hosting OpenClaw containers. Each customer gets a Docker container on a shared Hetzner VPS managed by Dokploy. Current image runs as root which is the main security gap. These changes get us to "ready for first paying customer."
Repo
MattByy/openclaw-root — branch main
Changes to make
1. Dockerfile — switch to non-root user + pin base image
The base image ghcr.io/phioranex/openclaw-docker:latest already has a node user. We want to run as node after all root operations are done.
Get the current digest of the base image and pin to it instead of :latest:
bashdocker pull ghcr.io/phioranex/openclaw-docker:latest
docker inspect --format='{{index .RepoDigests 0}}' ghcr.io/phioranex/openclaw-docker:latest
In the Dockerfile:

Change FROM ghcr.io/phioranex/openclaw-docker:latest to pinned digest
Keep USER root at the top (needed for apt/pip/npm/playwright installs)
Add USER node at the very end, after all COPY and RUN steps
Ensure /home/node and everything under /opt/clawmode is chown -R node:node before the USER switch
Add a HEALTHCHECK directive that curls http://127.0.0.1:18789/healthz

2. Entrypoint — make it work as non-root
The entrypoint is generated by n8n and mounted at /entrypoint.sh. We don't control n8n from this repo, BUT we control whether the image can support a non-root entrypoint. Things to verify/fix:

The entrypoint does chown root:root /home/node/.openclaw/.env + chmod 600. That breaks when running as node. We need to document that n8n's Config Builder should use chown node:node instead. Update Claude.md with this requirement.
The entrypoint does su -s /bin/sh node -c "..." which assumes root. If we switch to USER node, su won't work. Document in Claude.md that when the image runs as node, these su wrappers must be removed from entrypoint.
